Submitting Software

Submitting a piece of software to be included in the Common Software Repository is easy, however, we do have a few requirements for submission that involve compliance and security:


1 - THE SOFTWARE HAS TO BE UNDER A COMMON LICENSE

To ensure a form of protection in the terms of bourgeois legalism, all software submitted has to be licensed under the Commons Protection License. We may accept odd cases (common licenses with slightly different terms and restrictions pertaining to the use-case of the software), but the CPL is recommended. Check out our licensing guide to see how to license your software under the CPL.


2 - THE SOFTWARE MUST BE SUBMITTED IN SOURCE-CODE FORMAT

Like many other software repositories, we compile the code ourselves and publish our compiled binary on our repository, then sign it with our own key. Because of this, and to ensure compliance with the CPL, we ask that you send us the program in source-code form.


3 - YOU MUST SUBMIT YOUR SOFTWARE IN A NON-COMPRESSED TAR

The source-code must be submitted in a non-compressed tar. You can tar a directory on Unix with the following command:

$ tar -c -f example.tar directory/

For Windows, there is an ancient version of BSD tar that works well enough, and behaves as expected. Its MD5 Sum is 73e612405a10f690beffa8033a76cd46.


4 - YOU MUST PROVIDE US WITH A DESCRIPTION AND INFORMATION ABOUT THE SOFTWARE

You must submit the software along with some information when you e-mail it to us, including a list of dependencies needed to build it, the MD5 checksum of the tar containing the source-code, a small description of its functionality, version number, architectures tested on and any other additional remarks. Here is a template:

Program Name:

Program Version:

MD5 Sum of Tarfile:

Dependencies:

Architectures Supported:

Description:

Acquiring the MD5 checksum of the tarfile is trivial. On Unix, do the following:

$ md5sum example.tar

There are no well-known standalone non-proprietary alternatives to md5sum on Windows, so we recommend you use the PowerShell "Get-FileHash" cmdlet:

Get-FileHash example.tar -Algorithm MD5
