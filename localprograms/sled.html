RTC Silly Line Editor - sled
Because you were expecting a serious line editing tool.

But silly doesn’t mean bad. In fact, if you want a line editor that’s simple to use and can do basic tasks without a bothersome syntax, along with working basically everwhere (see compatibility section), sled is for you! With the bulk of the editor being made in 2 days while on a caffeine binge, there’s no better program to just edit a configuration file with.

Usage
The Silly Line Editor is invoked with sled -f [file]. It can be used to edit existing files, or to create new ones. Invoking the program in such a way will open the file and drop you into a prompt. For quick edits, a user may utilize the -c command argument to quickly invoke an editor action from the shell.

Command List
?h - Show copyright and further help;
?H - Show this text
?q - Quit the program;
?s - Save file;
?l - List file;
?L [begin] [end] - List file from line begin to line end;
?c [line] [command] - Pipe shell output to line;
?e [lineno] - Erase line at lineno;
?E [begin] [end] - Delete lines from begin to end
?w [lineno] [text] - Add new line after lineno containing text;
?p [offset] - Seek to byte to destructively insert;
?P - Seek to line to destructively insert.

So, as an example, if we wish to include a calendar into a file with a brief explanation of the month:

$ sled -f calendar.txt

'?q' for quitting, '?h' for help
?c 1 cal
I really liked the month of January!

If we want to do the same thing in the command line:

$ sled -f calendar.txt -c "?c 1 cal"
$ sled -f calendar.txt -c "I really liked the month of January, because Lula got elected."
